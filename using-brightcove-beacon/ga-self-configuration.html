Einführung

Der Konfigurationsprozess besteht aus folgenden Aufgaben:

  • Erstellen Sie vier Google Analytics-Properties.
  • Erstellen Sie für jede Property einen Satz benutzerdefinierter Dimensionen und einen benutzerdefinierten Messwert.
  • Erstellen Sie vier Google Tag Manager-Container.
  • Konfigurieren Sie Google Tag Manager für jeden Container.
  • Konfigurieren Sie Firebase, um die Analysesammlung mit iOS und Android zu unterstützen.
  • Installieren Sie Vorlagen in Google Data Studio, um Ihre Analysedaten anzuzeigen.

Notwendige Werkzeuge

Brightcove verwendet verschiedene Tools, um die Erfassung, Bearbeitung und Anzeige von Analysedaten mithilfe von Google Analytics zu implementieren. Diese Werkzeuge sind:

  • Google Analytics - Eine Webanalyseplattform, die von Google als Teil ihrer Marke Google Marketing Platform angeboten wird.
  • Google Tag Manager - Verstehen Sie zunächst, dass Tags Code-Schnipsel sind, die einer Site hinzugefügt werden, um Informationen zu sammeln und an Dritte zu senden. Tags werden aus zahlreichen Gründen verwendet, darunter Scroll-Tracking, Überwachung von Formulareinsendungen, Durchführung von Umfragen usw. Mit Google Tag Manager können Sie "Tags schnell verwalten und aktualisieren und schneller fundierte Entscheidungen treffen. Tag Manager lässt sich in alle Google- und Drittanbieter-Tags integrieren und gibt Ihnen die vollständige Kontrolle darüber, wann und wie Ihre Tags ausgelöst werden."
  • Feuerbasis - Firebase ist eine von Google entwickelte Plattform zum Erstellen von Mobil- und Webanwendungen. Firebase enthält die Firebase-Echtzeitdatenbank , eine in der Cloud gehostete Datenbank.
  • Google Data Studio - Google Data Studio ist ein "kostenloses Tool, das Ihre Daten in informative, einfach zu lesende, einfach zu teilende und vollständig anpassbare Dashboards und Berichte umwandelt."

Abrufen von Vorlagen und Konfigurationsdateien

Es gibt eine Reihe von Dateien/URLs, die Sie benötigen, um Ihr Analytics-Setup für Brightcove Beacon erfolgreich abzuschließen. Die Dateien/URLs werden in den folgenden Aufzählungszeichen beschrieben. Sie müssen sich an Ihren Brightcove-Onboarding-Manager wenden, um diese Dateien/URLs zu erhalten. Du wirst erhalten:

  • URLs zu drei vorgefertigten Google Analytics-Dashboard-Konfigurationen, jeweils eine für Mobilgeräte, Web und Smart-TV.
  • Vier JSON-Dateien, die jeden der vier in Google Tag Manager erstellten Container konfigurieren.
  • URLs zu drei vorgefertigten Google Data Studio-Vorlagen, jeweils eine für Mobilgeräte, Web und Smart-TV.

Anweisungen zum Installieren/Verwenden der einzelnen Dateien/URLs finden Sie in den entsprechenden Abschnitten weiter unten in diesem Dokument.

Sie müssen diese Dateien/URLs abrufen, BEVOR Sie mit dem Konfigurationsprozess beginnen.

Google Analytics konfigurieren

In diesem Abschnitt des Dokuments konfigurieren Sie Google Analytics, das die folgenden Aufgaben umfasst:

  • Erstellen Sie vier Google Analytics-Properties.
  • Erstellen Sie eine mobile Ansicht für die iOS- und Android-Eigenschaften
  • Erstellen Sie für jede Eigenschaft einen Satz benutzerdefinierter Dimensionen.
  • Erstellen Sie für jede Property einen einzelnen benutzerdefinierten Messwert.
  • Importieren Sie drei von Brightcove erstellte Dashboard-Konfigurationen, jeweils eine für Mobilgeräte, Web und Smart-TV.

Erstellen Sie vier Eigenschaften

Sie müssen diese vier Eigenschaften erstellen:

  • Beacon Android
  • Beacon iOS
  • Beacon STV (Smart-TV)
  • Beacon-Web

Die folgenden Schritte führen Sie durch die Erstellung der Beacon Android Eigentum. Sie müssen die anderen nach dem gleichen Verfahren erstellen.

  1. Melden Sie sich bei Ihrem Google Analytics-Konto an.
  2. Drücke den Administrator (Ausrüstung) Symbol unten links auf dem Bildschirm.
  3. Klicken Sie oben in der mittleren Spalte auf Eigenschaft erstellen.
  4. In dem Einrichtung der Unterkunft , liefern die Name des Anwesens , für die erste Eigenschaft verwenden Sie Beacon Android.
  5. Liefern Sie die Zeitzone melden.
  6. Liefern Sie die Währung.
  7. Drücke den Erweiterte Optionen anzeigen Verknüpfung.
  8. Anmachen Erstellen Sie eine Universal Analytics-Property.
  9. Geben Sie die URL Ihrer Website ein.
  10. Drücke den Nur eine Universal Analytics-Property erstellen Radio knopf.
  11. Stellen Sie sicher, dass Ihre Immobilieninformationen wie folgt ausgefüllt sind:
  12. Klicken Nächste.
  13. Vervollständigen Sie die Geschäftsinformationen Formular mit Ihren Daten.
  14. Klicken Schaffen.
  15. Lesen Sie alle gewünschten Details zu Ihrer neuen Immobilie.
  16. Klicken Sie auf den Zurück-Pfeil, der im folgenden Screenshot hervorgehoben ist:
  17. Direkt unter dem Eigenschaft erstellen Sie zuvor geklickt haben, klicken Sie auf den Abwärtspfeil, um sicherzustellen, dass Ihre Property korrekt erstellt wurde.
  18. Führen Sie die folgenden Schritte aus, um die drei anderen erforderlichen Eigenschaften zu erstellen:
    1. Beacon iOS
    2. Beacon STV
    3. Beacon WEB

Mobile Ansicht für iOS und Android erstellen

Für mobile Plattformen, iOS und Android, müssen Sie in jeder dieser Eigenschaften eine mobile Ansicht erstellen. Sie müssen die Schritte in diesem Abschnitt nicht ausführen, wenn Sie keine mobilen Apps haben.

  1. Klicken Sie auf das Dropdown-Menü (im Screenshot gelb hervorgehoben), um alle Ihre Google Analytics-Konten, -Properties und -Ansichten anzuzeigen.
  2. Wählen Sie aus der Dropdown-Liste mit drei Feldern Folgendes aus:
    • Ihr Brightcove Beacon-Konto
    • Die Beacon Android Eigentum
    • Die Alle Website-Daten Aussicht
  3. Drücke den Administrator (Ausrüstung) Symbol unten links auf dem Bildschirm.
  4. Klicken Sie oben in der dritten Spalte auf Ansicht erstellen.
  5. Füllen Sie die folgenden Schritte aus:
    1. Drücke den App Schaltfläche für den zu verfolgenden Datentyp.
    2. Geben Sie einen Namen für Ihre Ansicht an. Ein logischer Name wäre Alle mobilen Daten.
    3. Klicken Ansicht erstellen.
  6. Ändern Sie die Ansicht in die neu erstellte mobile Ansicht.
  7. Führen Sie die gleichen Schritte in diesem Abschnitt des Dokuments aus, um eine mobile Ansicht für die iOS-Property zu erstellen.

Stellen Sie bei der Arbeit mit Ihren Konten sicher, dass Sie bei der Auswahl eines Kontos/einer Immobilie/einer Anzeige Ihrer mobilen Eigenschaften die Alle mobilen Daten Ansicht und die Web- und STV-Konten verwenden die Standardeinstellung Alle Website-Daten , wie hier gezeigt:

Beachten Sie, dass die iOS- und Android-Ansichten haben Alle Website-Daten Ansichten, da diese beim Erstellen der Eigenschaft standardmäßig erstellt werden

Benutzerdefinierten Messwert und Dimensionen erstellen

Nun müssen Sie für jede Property einen Satz benutzerdefinierter Dimensionen und einen benutzerdefinierten Messwert erstellen.

Dieser Abschnitt wird wie folgt organisiert:

  • Es werden Schritte gezeigt, wie Sie eine einzelne benutzerdefinierte Dimension erstellen.
  • Es werden Schritte gezeigt, wie Sie einen einzelnen benutzerdefinierten Messwert erstellen.
  • Eine Tabelle zeigt die erforderlichen Maße für die Eigenschaften Beacon Android und Beacon iOS.
  • Eine Tabelle zeigt die erforderlichen Maße für die Eigenschaften Beacon STV und Beacon WEB.
  • Eine Tabelle zeigt die einzelne Metrik, die alle vier Eigenschaften haben müssen.

Anschließend müssen Sie die Dimensionen und den Messwert für jede der vier Eigenschaften erstellen.

Erstellen einer benutzerdefinierten Dimension

  1. Wählen Sie in der Konto-/Property-/Datenansicht-Auswahl eine Ihrer Properties aus, für die Sie die benutzerdefinierten Dimensionen erstellen möchten.
  2. Drücke den Administrator (Ausrüstung) Symbol unten links auf dem Bildschirm.
  3. Unten in der mittleren Spalte, unter Benutzerdefinierte Definitionen , klicken Benutzerdefinierte Abmessungen
  4. Klicken NEUE BENUTZERDEFINIERTE DIMENSION.
  5. Für die Name Eintreten Episodennummer.
  6. Von dem Umfang Dropdown, wählen Sie Schlag.
  7. Verlassen Sie die Dimension Aktiv.
  8. Klicken Schaffen.
  9. Klicken Sie im Beispielcodebildschirm auf Fertig.
  10. Sie sehen Ihre neu erstellte Dimension in einer Liste, wie hier gezeigt:
    Entscheidend ist, dass die Index stimmt mit dem Dimensionsnamen überein, wie in den folgenden Tabellen gezeigt. Erstellen Sie dazu einfach die Maße in der in den Tabellen angegebenen Reihenfolge.
  11. Um eine weitere Dimension zu erstellen, klicken Sie auf NEUE BENUTZERDEFINIERTE DIMENSION.

Erstellen eines benutzerdefinierten Messwerts

  1. Wählen Sie eine Ihrer Properties aus der Konto-/Property-/Datenansicht-Auswahl aus, für die Sie den benutzerdefinierten Messwert erstellen möchten.
  2. Drücke den Administrator (Ausrüstung) Symbol unten links auf dem Bildschirm.
  3. Unten in der mittleren Spalte, unter Benutzerdefinierte Definitionen , klicken Benutzerdefinierte Messwerte
  4. Klicken NEUE BENUTZERDEFINIERTE METRIK.
  5. Für die Name Eintreten Gesehene Zeit.
  6. Von dem Umfang Dropdown, wählen Sie Schlag.
  7. Für die Formatierungstyp , auswählen Zeit.
  8. Verlassen Sie die Dimension Aktiv.
  9. Klicken Schaffen.
  10. Klicken Sie im Beispielcodebildschirm auf Fertig.
  11. Sie sehen Ihre neu erstellte Metrik in einer Liste.
Dies ist der einzige benutzerdefinierte Messwert, den Sie für jede der vier Eigenschaften erstellen müssen.

Konfigurationsdateien importieren

Führen Sie die folgenden Schritte aus, um die Konfigurationsdateien für Ihre Container zu importieren:

  1. Von dem Arbeitsplatz , klicken Administrator.
  2. Überprüfen Sie, welchen Container Sie gerade konfigurieren, und klicken Sie dann auf Container importieren.
  3. Klicken Containerdatei auswählen , und wählen Sie die richtige JSON-Datei für diesen Container aus.
  4. Zum Arbeitsbereich auswählen , klicken Bestehende , dann aus dem sich öffnenden Dialog wählen Standardarbeitsbereich.
  5. Zum Wählen Sie eine Importoption , drücke den Überschreiben Radio knopf.
  6. Stellen Sie sicher, dass Sie Informationen zu Tags, Triggern, Variablen und Vorlagen sehen.
  7. Klicken Bestätigen Sie , und Sie werden zurück zum Arbeitsplatz..
  8. In dem Arbeitsplatz , klicke auf Stichworte , Löst aus und Variablen um zu bestätigen, dass der Import erfolgreich war.
  9. Sie müssen nun für jeden Container die richtige JSON-Datei importieren.

Tracking-ID eingeben

Sie kopieren nun die Tracking-ID für jede Property aus Google Analytics und fügen sie dann in den entsprechenden Container ein.

  1. Melden Sie sich bei Google Analytics an.
  2. Verwenden Sie die Konto-/Property-Auswahl oben auf der Seite, um die Beacon Android Eigentum.
  3. Klicken Administrator.
  4. Von dem Eigentum Spalte erster Klick Tracking-Informationen dann Verfolgungscode.
  5. Kopiere das Tracking ID.
  6. Kehren Sie zum Google Tag Manager zurück.
  7. Stellen Sie sicher, dass Sie die bearbeiten Beacon Android Container.
  8. Klicken Variablen aus der linken Seitennavigation.
  9. Drücke den Verfolgungscode Variable.
  10. Fügen Sie die kopierte Tracking-ID von Google Analytics in das Wert Formularfeld.
  11. Sie müssen nun die entsprechende Tracking-ID in das Verfolgungscode variabel in jedem Container.

Veröffentlichen Sie jeden Container

Später in diesen Schritten veröffentlichen Sie Ihre Container. Für iOS und Android müssen Sie Konfigurationsdateien herunterladen und Ihr App-Entwicklungsteam diese Dateien der jeweiligen App hinzufügen lassen.

Der letzte Schritt, den Sie im Google Tag Manager ausführen müssen, besteht darin, jeden Container zur Veröffentlichung einzureichen. Die folgenden Schritte führen Sie durch diesen Prozess:

  1. Kehren Sie zu Ihrem Arbeitsbereich zurück.
  2. Notieren Sie den Container, in dem Sie sich befinden.
  3. Klicken Sie oben rechts in der Benutzeroberfläche auf einreichen.
  4. Seien Sie sicher, dass Version veröffentlichen und erstellen Schaltfläche ausgewählt ist.
  5. Wenn Sie möchten, fügen Sie einen Versionsnamen und eine Versionsbeschreibung hinzu.
  6. Klicken Veröffentlichen.
  7. Wenn Sie keinen Versionsnamen und keine Beschreibung angegeben haben, werden Sie erneut gefragt. Überspringen Sie, wenn Sie sicher sind, dass Sie diese nicht möchten.
  8. Stellen Sie sicher, dass die zu Ihrem Container zurückgegebenen Informationen korrekt sind.
  9. Für iOS- und Android-Container: Klicken Herunterladen oben rechts auf der Benutzeroberfläche, um die zugehörige Datei herunterzuladen. Diese Datei muss an Ihren Onboarding-Manager gesendet werden.
  10. Für iOS- und Android-Container: Wenden Sie sich an Ihren Onboarding-Manager und senden Sie ihm diese heruntergeladenen Dateien zusammen mit dem Screenshot Ihrer Container und den entsprechenden IDs. Die Dateien werden Ihren entsprechenden Apps hinzugefügt.
  11. Stellen Sie sicher, dass alle Container veröffentlicht sind.
Um es noch einmal zu wiederholen: An dieser Stelle müssen Sie die heruntergeladenen Dateien zusammen mit dem Screenshot senden, der jeden Containernamen und die zugehörigen Container zeigt GTM ID, an Ihren Onboarding-Manager. Diese Dateien werden dann an die technischen Mitarbeiter von Brightcove Beacon weitergeleitet, um Ihre Apps zu konfigurieren.

  8. Seien Sie sicher, dass Sie zum . zurückkehren Projektübersicht Seite. Beachten Sie auch die Schaltflächen zum Erstellen von Apps für iOS und Android (die roten Pfeile zeigen ihre Position an).

Um die Schritte abzuschließen, benötigen Sie einen iOS-Entwickler zur Unterstützung. Das folgende Google-Dokument enthält Details zu den Schritten:
    • Firebase zu Ihrem iOS-Projekt hinzufügen.

Um die Schritte abzuschließen, benötigen Sie einen Android-Entwickler zur Unterstützung. Das folgende Google-Dokument enthält Details zu den Schritten:
    • Firebase zu Ihrem Android-Projekt hinzufügen.
