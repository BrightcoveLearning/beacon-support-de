    Implementierung von Live-SSAI

    In diesem Thema erfahren Sie, wie Sie serverseitige Live-Anzeigenkonfigurationen in Brightcove Beacon verwalten und erstellen.

    Übersicht

    Brightcove Beacon unterstützt serverseitige Anzeigeneinfügung (SSAI) bei Live-Events, sodass serverseitige Anzeigen während eines Live-Streams angefordert und angezeigt werden können.

    Eine allgemeine Sicht auf den Prozess ist:

    1. Erstellen Sie eine Anzeigenkonfiguration in Studio.
    2. Erstellen Sie einen Fill Slate für Ihr Live-Event.
    3. Erstellen Sie ein Live-Event mit aktivierter SSAI.
    4. Veröffentlichen Sie das Live-Event auf Brightcove Beacon.
    Dieses Dokument ist eine Ergänzung zur Dokumentation zum Streaming a Live Event, um Sie dabei zu unterstützen, SSAI zu Ihren Live-Events hinzuzufügen. Ausführliche Informationen zum Erstellen einer Veranstaltung finden Sie im Abschnitt Erstellen eines Live-Events in Brightcove Beacon.

    SSAI bietet diese Hauptfunktionen während Live-Events:

    • Da Anzeigen auf der Serverseite in den Live-Videostream eingefügt werden, sind sie nicht von Werbeblockern betroffen.
    • Anzeigen werden an Cue-Points eingefügt, die von Ihrem Encoder gesendet werden, oder Sie können mit dem Live-Modul Control Room einen sofortigen Cue-Point erstellen.
    • Sie können Füllschiefer so konfigurieren, dass sie ungenutzte Werbezeiten ausfüllen.

    Konfigurieren der serverseitigen Anzeigeneinstellungen

    Hier erstellen Sie eine SSAI-Anzeigenkonfiguration, die als eine Reihe von Anzeigen verwendet wird, die in Ihrem Brightcove Beacon Live-Event angezeigt werden.

    1. Melden Sie sich bei Studio an.
    2. Klicken Sie im Navigationsheader auf ADMIN und wählen Sie Serverseitige Anzeigeneinstellungenaus. Eine Liste aller Anzeigenkonfigurationen wird angezeigt.
    3. Um eine neue Anzeigenkonfiguration zu erstellen, klicken Sie auf die Schaltfläche Anzeigenkonfiguration erstellen .
      Serverseitige Live-Anzeigenkonfigurationen können auch mit der Live-API verwaltet werden. Weitere Informationen finden Sie unter Brightcove Live API: Serverseitige Anzeigeneinfügung (SSAI).
    4. Wählen Sie im Dropdown-Menü Live als Typ aus.
    5. Vergewissern Sie sich, dass Sie das Formular „Serverseitige Anzeigeneinstellungen“ sehen.
    6. Geben Sie Ihrer Anzeigenkonfiguration einen Namen.
    7. Wählen Sie die Antwort auf den Adserveraus:
      • GROSSES
      • DFP (Anzeigenregeln)
      • Smartes XML
      Bitte erkundigen Sie sich bei Ihrem Ad Operations Manager nach dem richtigen Wert.
    8. Geben Sie das Anzeigen-Tag (Server-URL) ein.
    9. Klicken Sie auf den Button Speichern .

    Konfigurieren eines Fill Slate

    Füllschiefer sind Videos, die angezeigt werden, wenn Lücken in einer Live-Stream-Werbeunterbrechung bestehen, die nicht mit einer dynamisch bereitgestellten Anzeige gefüllt werden kann. Wenn die angeforderte Anzeige kürzer ist als die Werbepause, wird die Füllungsliste angezeigt, sodass die Zuschauer keinen leeren Player sehen.

    Sie sollten mindestens einen Fill Slate konfigurieren, bevor Sie eine serverseitige Anzeige anfordern.

    Um einen Füll-Slate hinzuzufügen, gehen Sie folgendermaßen vor:

    1. Klicken Sie im Studio-Header auf den Link ADMIN > Live-Einstellungen .
      Sie müssen Kontoadministrator sein, um auf das Menü ADMIN zugreifen zu können.
    2. Klicken Sie auf + Füllen Slate hinzufügen. Sie sollten das folgende Formular sehen:
    3. Geben Sie einen Namen für Ihr Fill Slate ein.
    4. Geben Sie die Quell-URL für den Slate ein.
    5. Klicken Sie auf Save. Der Aufwangsprozess für den Schiefer wird beginnen.

    Um eine Fülltafel zu löschen, klicken Sie auf die Schaltfläche „Löschen“ () neben dem Füllfeld.

    Erstellen eines Live-Events, das SSAI unterstützt

    Damit ein Live-Event SSAI (serverseitige Anzeigeneinfügung) verwenden kann, muss die Einstellung Serverseitige Anzeigeneinfügung (SSAI) aktivieren ausgewählt werden, wenn das Ereignis erstellt wird, und eine zuvor erstellte Fill-Slate sollte für die Wiedergabe ausgewählt werden.

    Befolgen Sie diese Schritte, um ein Event zu erstellen, das SSAI-Anzeigen in Brightcove Beacon unterstützt:

    1. Öffnen Sie das Live-Modul in Studio.
    2. Klicken Sie auf der Seite „Ereignisse verwalten “ auf „ Ereignis erstellen“.
    3. Geben Sie Ihre Veranstaltungsdetails ein. Vollständige Informationen zum Erstellen eines Live-Events mit dem Live-Modul finden Sie unter Erstellen und Verwalten von Live-Events mit dem Live-Modul.
    4. Klicken Sie auf ERWEITERTE OPTIONEN um die erweiterten Optionen zu erweitern.
    5. Aktivieren Sie Serverseitige Anzeigeneinfügung aktivieren (SSAI). Dies ermöglicht das einfügen serverseitiger Anzeigen.
    6. Wählen Sie ein Fill Slate. Die Fill Slate wird angezeigt, wenn Lücken in einer Live-Stream-Werbeunterbrechung bestehen, die nicht mit einer dynamisch bereitgestellten Anzeige gefüllt werden kann.
    7. Wenn die Einstellung „ Ad Audio “ aktiviert ist, werden die Audiopegel in Anzeigen basierend auf der festgelegten Anpassung normalisiert. Dies kann verwendet werden, um sicherzustellen, dass Anzeigen nicht wesentlich lauter sind als der Inhalt. Brightcove empfiehlt, diese Option zu wählen.
    8. Klicken Sie auf Ereignis erstellen
    9. Die Control Room-Seite wird mit Details zum Live-Ereignis geöffnet, einschließlich des Streaming-Endpunkts (RTMP-URL) und des Stream-Namens. Diese Werte werden vom Encoder verwendet.
    10. Konfigurieren Sie Ihren Encoder und starten Sie den Live-Stream.
      Um mit dem Streaming auf der Live-Plattform zu beginnen, benötigen Sie einen Encoder. Informationen zur Verwendung eines Encoders finden Sie in der folgenden Dokumentation:

      Schritt für Schritt: Verwenden des Live-Moduls mit Telestream Wirecast

      Schritt für Schritt: Verwenden des Live-Moduls mit Open Broadcaster Software (OBS)

    11. Standardmäßig zeigt der Control Room-Vorschau-Player den werbefreien Stream an. Anzeigen werden nicht im Control Room geschaltet, es sei denn, eine Live-Anzeigenkonfiguration ist ausgewählt. Verwenden Sie das Dropdown-Menü unten für den Player, um eine Anzeigenkonfiguration auszuwählen.
    12. (Optional) Klicken Sie auf die Plus-Schaltfläche () , um eine zusätzliche Anzeigenkonfiguration auszuwählen. Dadurch wird auf der Control Room-Seite ein weiterer Vorschau-Player hinzugefügt.

      Klicken Sie auf die Lautstärketaste unter dem Vorschau-Player, um zu steuern, welcher Player-Audiostream aktiv ist.

    Anfordern einer Anzeige

    Befolgen Sie diese Schritte, um eine Anzeige im Live Control Roomanzufordern:

    1. Klicken Sie auf Gehe zu Ad und wählen Sie aus der Dropdown-Liste die Länge Ihres gewünschten Werbeblendes aus.
    2. Klicken Sie auf Einfügen , um einen Werbeblock in den Livestream einzufügen. Wenn die Anzeige nicht die gesamte Werbepause ausfüllt, wird die Füllungsliste (die ausgewählt wurde, als das Ereignis erstellt wurde) angezeigt.

    Anzeigen werden im Vorschauspieler des Control Room und in Spielern angezeigt, die mit dem Live-Modul veröffentlicht werden. Dieses Beispiel zeigt 2 Anzeigen, die im Control Roomgeschaltet werden.

    Veröffentlichen einer Live-Veranstaltung in Brightcove Beacon

    Gehen Sie folgendermaßen vor, um Ihr Live-Event mit serverseitigen Anzeigen in Brightcove Beacon zu veröffentlichen:

    1. Klicken Sie im Live Control Room auf Veröffentlichen und Einbetten in der linken Navigationsleiste.
    2. Wählen Sie Player-Konfigurationaus.
    3. Wählen Sie in den Live-Player-Optionen einen verfügbaren Spieleraus.
    4. Wähle Deine Anzeigenkonfiguration zuvor in der erstellt Serverseitige Anzeigeneinstellungen.
    5. Nachdem Sie Ihre Verfügbare Player- und Anzeigenkonfiguration ausgewählt haben, wird eine neue Player-URL generiert. Kopieren Sie den ausgewählten Teil der URL wie im Bild gezeigt, beginnend mit dem Wort live. Beispiel: live.xxxxxxx
    6. Rufen Sie die Brightcove Beacon Platform auf.
    7. Klicken Sie in Brightcove Beacon im oberen Menü auf den Tab Werbung .
    8. Klicken Sie auf die Schaltfläche „ Neue Konfiguration hinzufügen “.
    9. Vergewissern Sie sich, dass Sie das Formular für neue Werbung sehen.
    10. Geben Sie Ihrer neuen Werbekonfiguration einen Namen.
    11. Fügen Sie in die Live-Anzeigenkonfig-ID: die Player-URL-ID ein, die Sie gerade von der Live-Plattform kopiert haben, zum Beispiel: live.xxxxx.
    12. Wählen Sie im Menü Anbieter die Option Brightcove SSAIaus.
    13. Klicken Sie auf die Schaltfläche „ Neue Werbung erstellen “.
    14. Klicken Sie auf den oberen Menü-Tab Events und erstellen oder wählen Sie Ihr Live-Event aus. Weitere Informationen zum Erstellen eines Brightcove Beacon Live-Events finden Sie im Dokument Streaming a Live Event .
      Damit das Live-Event in Brightcove Beacon erfolgreich streamen kann, müssen die Job-ID und die Stream-Video-ID mit den IDs des aktuellen Live-Ereignisses übereinstimmen.
    15. Klicken Sie in Ihrem Brightcove Beacon Live-Event auf die Registerkarte „ Rechte und Planung “.
    16. Wählen Sie im Dropdown-Menü Rechteart die Option Werbung - AVODaus.
    17. Wählen Sie im unteren Menü der Anzeigenkonfiguration Ihre zuvor erstellte Anzeigenkonfiguration aus.
    18. Klicken Sie auf die Schaltfläche „ Ereignis aktualisieren “.
    19. Klicken Sie oben rechts auf dem Bildschirm auf die Schaltfläche Extras (Schraubenschlüssel). Leeren Sie dann den Cache.
    20. Öffnen Sie Ihre Brightcove Beacon App und wählen Sie Ihre Veranstaltung aus (die Ereignisnamen und Bilder auf dem Bild dienen nur zum Beispiel).
    21. Klicken Sie in Ihrer Brightcove Beacon-App auf die Schaltfläche „ Abspielen “. Sie sollten jetzt in der Lage sein, Ihr Live-Event-Streaming mit Anzeigen von SSAI aus Video Cloud eingefügt zu sehen.

    Sobald Ihr Event endet, können Sie es ganz oder teilweise als Clip speichern, der zu einem Video-Asset in Video Cloud oder einem abspielbaren Stream wird, der in Brightcove Beacon angezeigt werden kann. Die Anzeigen des Live-Events werden nicht gespeichert, aber die Cue-Points, von denen die Anzeigen übertragen wurden, werden registriert, damit Sie neue Anzeigen in einem späteren VOD Ihres aufgezeichneten Live-Events hinzufügen können.

    Weitere Informationen zum Erstellen eines Clips finden Sie im Abschnitt Erstellen eines Clips aus Ihrem Live-Event im Dokument Streaming a Live Event .
