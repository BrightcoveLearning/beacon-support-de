Einführung

Brightcove Cloud Playout ist ein Video Cloud Studio-Modul, mit dem Sie eine geplante Wiedergabe von Elementen aus Ihrem VOD-Katalog in Kombination mit Live-Feeds programmieren und als linearen Kanal mit der Zuverlässigkeit und Skalierbarkeit von Brightcove Live streamen können.

Sie können den Cloud Playout-Kanal dann als Beacon-Kanal verwenden, indem Sie beim Erstellen eines Beacon-Kanal. Ein großer Vorteil der Verwendung von Cloud Playout-Kanälen besteht darin, dass Sie keinen EPG-Anbieter eines Drittanbieters suchen und integrieren müssen, da ein EPG-Dienst in Cloud Playout integriert ist.

Cloud Playout bietet eine leistungsstarke End-to-End-Lösung für Kanäle mit geplanten Programmen, darunter:

  • 24x7 lineare Kanäle
  • Pop-Up / virtuelle Kanäle
  • Simulierte Live-Ereignisse

Cloud Playout hilft Ihnen, das Engagement des Publikums zu steigern und neue Umsatzmöglichkeiten durch ein sendeähnliches Live-TV-Erlebnis mit vorab aufgezeichneten Inhalten zu schaffen. Verwenden Sie vorhandene VOD-Assets, um geplante Live-Kanäle zu erstellen.

Merkmale

Zu den Funktionen von Cloud Playout gehören:

  • Importieren Sie VOD-Assets aus Ihrer Video Cloud-Medienbibliothek.
  • Wechseln Sie zwischen VOD-Assets und geplanten Live-Feeds.
  • Reagieren Sie in Echtzeit auf kurzfristige Fahrplanänderungen einfach, schnell und zuverlässig.
  • Monetarisieren Sie Ihren Kanalinhalt mit serverseitiger Anzeigeneinfügung (SSAI).
  • Konfigurieren Sie Ihren Kanal, um Ihr Programm zu wiederholen.
  • Optimieren Sie die Wiedergabe nach Gerät.
  • Ziehen und ablegen, um den Programmplan jederzeit neu zu ordnen, sogar im Status „Läuft“.
  • API zum Abrufen des EPG (elektronischer Programmführer) für einen Kanal

Workflow-Übersicht

Es gibt zwei Teile, um einen Cloud Playout-Kanal zu erstellen und ihn zu Ihrer Beacon-App hinzuzufügen:

  1. In Video-Cloud , erstellen Sie den Kanal.
  2. In Leuchtfeuer , erstellen Sie einen neuen Beacon-Kanal, der den Cloud Playout-Kanal als Quelle verwendet.

Einschränkungen

Es gelten die folgenden Annahmen und Einschränkungen.

  • Ihre Videos müssen mit einem Dynamic Delivery-Profil aufgenommen oder erneut transkodiert werden und muss eine MP4-Version haben
  • Die maximale Eingangsbitrate und Auflösung, die für Cloud Playout unterstützt wird, beträgt 20 MBPS/1080p. Dies gilt sowohl für VOD-Inhalte als auch für Live-Eingaben.
  • Brightcove empfiehlt die Verwendung von Videos mit einer Dauer von mindestens 2 Minuten, um einen korrekten Übergang zwischen Videos und die Genauigkeit des elektronischen Programmführers zu gewährleisten
  • Zu Videos hinzugefügte Untertitel (Texttracks) werden derzeit nicht in Cloud Playout-Kanälen angezeigt. Die Unterstützung für Untertitel wird in einer zukünftigen Version hinzugefügt.
  • Die Mindestdauer für Live-Events beträgt 5 Minuten.
  • Jedes Live-Ereignis in Cloud Playout teilt eine 1:1-Beziehung mit einem Video Cloud-Remote-Asset. Dieses Remote-Asset kann nicht in anderen Kanälen wiederverwendet werden. Wenn das Live-Ereignis aus dem Kanal entfernt wird, wird das zugehörige Remote-Asset automatisch gelöscht. Wenn ein Kanal gelöscht wird, werden auch alle zugehörigen Remote-Assets automatisch gelöscht.
  • Videos geteilt auf das Konto eines anderen Kontos kann derzeit nicht zu einem Cloud Playout-Kanal hinzugefügt werden.
  • Wenn gerade ein Live-Event abgespielt wird, aber kein aktiver Feed verfügbar ist, wird der Slate-Inhalt in einer Schleife abgespielt, bis der Live-Feed fortgesetzt wird oder das Event endet. Es wird eine Verzögerung von ungefähr 30 Sekunden geben, bevor die Slate MP4 abgespielt wird.
  • Wenn das Video Cloud-Konto für die geografische Beschränkung konfiguriert ist, wird der Kanal Ausgaberegion sollte in einer Region erstellt werden, die mit der Richtlinie für geografische Beschränkungen von Video Cloud kompatibel ist.
  • Für die beste Erfahrung empfiehlt Brightcove die Verwendung des Google Chrome-Browsers.
