Einführung

Im ersten Teil dieses Dokuments wird beschrieben, wie Brightcove Beacon allgemeine Daten wie Videos, Playlists und Live-Ereignisse erhält. Im letzten Abschnitt wird ausführlich beschrieben, wie Open ID Connect (OIDC) und Berechtigungen verwendet werden, um Brightcove Beacon zum Abrufen von Daten zu autorisieren.

Videoaufnahme und benutzerdefinierte Felder

Video-Assets müssen zuerst in Video Cloud aufgenommen werden, bevor Brightcove Beacon sie verwenden kann. Je nachdem, wie die Videos verwendet werden, zum Beispiel in einer Serie oder für Werbung, kann Video Cloud Benutzerdefinierte Felder kann verwendet werden, um das Video mit diesen speziellen Anforderungen zu verknüpfen. Die Erstellen benutzerdefinierter Video Cloud-Felder Das Dokument beschreibt detailliert, wie Sie erforderliche benutzerdefinierte Felder erstellen und diesen Feldern Werte zuweisen.

Sobald sich die Videos in Video Cloud befinden, kann Brightcove Beacon Informationen zu den Videos wie Metadaten, Bilder und Speicherort aus dem Video Cloud-Katalog abrufen. Die Videos verbleiben tatsächlich in Video Cloud und werden von der Video Cloud-Wiedergabebereitstellungsinfrastruktur bereitgestellt. In der Brightcove Beacon-Benutzeroberfläche können Sie die Video Cloud-Videos manuell aufnehmen' Informationen oder warten Sie auf einen Hintergrundprozess, um dies zu tun.

Wiedergabelisten

Zusätzlich zum Erstellen von Video-Assets in Video Cloud wird empfohlen, dass Sie Ihre Video-Wiedergabelisten auch in Video Cloud erstellen. Sie können in Video Cloud zwei Arten von Wiedergabelisten erstellen: Handbuch und Clever. Ausführliche Informationen zum Implementieren von Wiedergabelisten in Video Cloud finden Sie im Erstellen und Verwalten von Wiedergabelisten dokumentieren.

Wenn Sie eine Playlist erstellen möchten, die andere Assets als Videos enthält, z. B. Serien oder Live-Ereignisse, muss diese Playlist in Brightcove Beacon erstellt werden.

Kanäle und Live-Events

Für Kanäle und Live-Ereignisse müssen die Live-Jobs in der Video-Cloud erstellt und dann manuell in Brightcove Beacon konfiguriert werden. Für Details siehe die Ein Live-Event streamen und Einen Kanal erstellen Unterlagen.

Serien, Staffeln und Episoden

Für episodische Daten müssen Sie sowohl Brightcove Beacon als auch Video Cloud verwenden. Sie müssen zuerst die Serie und die entsprechenden Staffeln in Brightcove Beacon erstellen. Die eigentliche Videoepisode wird in Video Cloud aufgenommen und dann werden benutzerdefinierte Felder verwendet, um das Video der richtigen Serie/Staffel zuzuordnen.

Ausführliche Informationen finden Sie im Serien/Jahreszeiten/Episoden erstellen dokumentieren.
