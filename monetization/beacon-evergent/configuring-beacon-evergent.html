Einführung

Die Konfiguration von Beacon und Evergent für die Zusammenarbeit erfordert einige Einstellungen in beiden Systemen. In einigen Fällen müssen Sie Informationen von einem System in das andere kopieren oder Informationen genau von einem System auf das andere replizieren. Wenn Sie beide Systeme nebeneinander in separaten Browserfenstern öffnen können, wird es einfacher, die folgenden Schritte erfolgreich auszuführen.

Leuchtfeuer Teil 1

Für diesen Teil benötigen Sie eine konfigurierte Beacon CMS (Customer Management System)-Instanz.

  1. Konfigurieren Sie Evergent im Beacon CMS als Geschäft.
  2. Auf der Einstellungen > Standort Seite für den Store, legen Sie die Länder fest, in denen der Service verfügbar sein wird.
  3. Auf der Einstellungen > Währungen Seite für den Shop, definieren Sie die Währungen für jedes Land.

Evergent Teil 1

Für diesen Teil müssen Sie BUs (Business Units) im Evergent Go Portal einrichten. Jede Evergent BU wird einem bestimmten Land oder einer bestimmten Region der Welt zugeordnet, und diese und die damit verbundenen Währungen sollten mit Ihrer Eingabe in das Beacon CMS übereinstimmen.

  1. Konfigurieren Sie die Währung der BU.
  2. Konfigurieren Sie das Land, in dem sich die BU befindet
  3. Konfigurieren Sie die Zahlungsgateways und Zahlungsmethoden für die BU:
    • Kreditkarte Debitkarte
    • E-Wallets
    • In-App-Käufe (iOS, Google Play, Amazon Billing, Roku Pay)

Leuchtfeuer Teil 2

Evergent BUs-IDs und Länderzuordnung müssen pro Beacon-Kunde konfiguriert werden. Dies erfolgt im Monetarisierungs-Zahlungsdienst, einem Back-End-Dienst, der dem Playback-Team gehört. (Der Plan ist, dass diese Konfiguration bis Ende Q1 21 auf Jackie Pages verschoben wird). Diese Konfigurationen sollten mit einem bestimmten Video Cloud-Kundenkonto verknüpft sein, das für Beacon verwendet wird.

Leuchtfeuer Teil 4

Ein Evergent Salz für diesen speziellen Beacon-Kunden muss der Monetarisierungs-Zahlungsdienst konfiguriert werden.

Leuchtfeuer Teil 5

Der Monetarisierungs-Zahlungsdienst generiert einen "Beacon_Monetization_Api"-Endpunkt (beacon.monetization.api.brightcove.com), der in die Beacon-CMS-Instanz des Kunden eingegeben werden muss. Dies sollte auf der Einstellungsseite unter "Payment Gateway Configuration" für Evergent eingegeben werden.

Leuchtfeuer Teil 6

  1. Erstellen Sie SVOD-Pläne auf der Registerkarte Beacon CMS Commerce. Definieren Sie die Monetarisierungsrichtlinien:
    1. Abrechnungshäufigkeit
    2. Probezeit
    3. Verlängerungsrichtlinie
    4. Wählen Sie aus, in welchen Stores der Plan angezeigt wird:
      • Evergent
      • App-Stores (iOS, Google Play, Amazon, Roku)
    5. Legen Sie die Geräte und Standorte fest.
    6. Legen Sie die Verfügbarkeitsdaten fest.
    7. Konfigurieren Sie die App-Stores:
      1. Konfigurieren Sie die Verfügbarkeitsdaten.
      2. Konfigurieren Sie die Geräte.
      3. Geben Sie den Preis und die Währungen ein.
      4. Erstellen Sie die Store-IDs. Diese ID muss kopiert und in die Produkt-Sku in den App Stores eingefügt werden. Außerdem muss es im Evergent Go für den passenden SVOD-Plan eingetragen werden.

Evergent Teil 2

Erstellen Sie in Evergent Go passende SVOD-Pläne. Es ist wichtig, dass die Monetarisierungsrichtlinien mit denen übereinstimmen, die im CMS von Beacon eingegeben wurden. Beachten Sie auch, dass Sie mehrere Evergent-BUs verwenden müssen, wenn Sie SVOD-Pläne in verschiedenen Währungen verkaufen möchten. Die Pläne in jeder BU werden Beacon mithilfe von SKUs zugeordnet, die im „Evergent Store“ im Beacon CMS eingegeben werden.

Leuchtfeuer Teil 7

Konfigurieren Sie den Beacon Evergent Store für Ihre SVOD-Pläne:

  1. Konfigurieren Sie die Verfügbarkeit.
  2. Geräte konfigurieren (derzeit nur Web).
  3. Konfigurieren Sie den Preis und die Währungen. Die Währungen sollten den Plänen in den spezifischen BUs entsprechen, die diese Währung verwenden. Wenn Sie beispielsweise einen Plan haben, den Sie in der EU mit Euro verkaufen, sollte die Evergent-ID von der EU Evergent BU stammen. In das Evergent-ID-Feld geben Sie die Evergent-SKU ein.
