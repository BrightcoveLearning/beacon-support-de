Erstellen benutzerdefinierter Video Cloud-Felder

In diesem Thema sehen Sie die benutzerdefinierten Video Cloud-Felder, die für die Synchronisierung mit Brightcove Beacon erforderlich sind.

Einführung

Videos, die aus Video Cloud in Brightcove Beacon importiert werden, müssen über einen erweiterten Satz von Metadaten verfügen, die für die ordnungsgemäße Verwendung in Brightcove Beacon definiert sind. In diesem Dokument erfahren Sie, wie Sie in Studio neue benutzerdefinierte Felder erstellen. Außerdem werden die erforderlichen Felder für Brightcove Beacon definiert.

Benutzerdefinierte Felder erstellen

Gehen Sie folgendermaßen vor, um benutzerdefinierte Felder zu erstellen.

  1. Melden Sie sich bei Video Cloud Studio an.
  2. Klicken Sie im Navigationskopf auf ADMINISTRATOR und dann Videofelder. Beachten Sie, dass Sie ein Kontoadministrator sein müssen, um auf das ADMIN-Menü zugreifen zu können.
  3. Auf der Seite Videofelder werden Ihre benutzerdefinierten Felder und alle standardmäßigen Videometadatenfelder angezeigt, wie im folgenden Screenshot gezeigt:
  4. Klicken Benutzerdefiniertes Feld hinzufügen.

  5. Geben Sie ein Anzeigename. Dies erscheint als Beschriftung für das Feld in den Modulen Upload und Medien. Ein Interner Name wird automatisch für das Feld basierend auf dem Anzeigenamen erstellt.

    Notiz: Die für den erfolgreichen Import in Brightcove Beacon zu verwendenden Anzeigenamen werden alle weiter unten in diesem Dokument angegeben.
  6. Wählen Sie a Typ für das Feld.
    • Text - Der Benutzer kann bis zu 1024 Einzelbyte-Zeichen (oder 512 Doppelbyte-Zeichen) in das Feld eingeben.
    • Aufführen - Der Benutzer wählt aus einer Liste von Werten aus. Listen können nicht mehr als 150 mögliche Werte haben und jeder Wert in der Liste darf nicht länger als 100 Zeichen sein.
  7. Geben Sie ein Beschreibung. Das Beschreibungsfeld wird als Hinweis in der Studio-Benutzeroberfläche angezeigt.
  8. Wenn das benutzerdefinierte Feld erforderlich ist, aktivieren Sie Machen Sie dieses Feld als erforderlich.

  9. Wenn Ihr Feldtyp ist Aufführen , geben Sie eine durch Kommas getrennte Werteliste ein und klicken Sie dann auf Zur Liste hinzufügen , wie im folgenden Screenshot gezeigt.

    Anmerkungen:
    1. Die Werte dürfen keine Anführungszeichen enthalten (Beispiel: Tiere, Vögel, Fische).
    2. Kein Wert darf ein Komma enthalten - alle Kommas werden als Trennzeichen für verschiedene Werte behandelt.
  10. Klicken Feld speichern um die Änderungen zu speichern. Klicken Speichern und weitere hinzufügen , um das Feld zu speichern und ein weiteres Feld zu erstellen.

Von Ihnen erstellte benutzerdefinierte Felder sind sofort verfügbar und wird sichtbar sein im Medienmodul.

Es können bis zu 10 benutzerdefinierte Felder erstellt werden. Wenn Sie mehr als 10 Felder benötigen, klicken Sie auf Benötigen Sie weitere Felder? Link auf der Seite Videofelder, um den Brightcove-Kundensupport mit einer Anfrage zu kontaktieren. Publisher sind auf 50 benutzerdefinierte Felder beschränkt.

Listenwerte bearbeiten

Sie können einem benutzerdefinierten Feld vom Listentyp zusätzliche Werte hinzufügen. Klicken Sie auf den Link für den benutzerdefinierten Feldnamen, fügen Sie die Werte hinzu und klicken Sie dann auf Zur Liste hinzufügen. Wenn Sie fertig sind, klicken Sie auf Feld speichern.

Listenwerte löschen

Sie können vorhandene Werte aus einem benutzerdefinierten Feld des Listentyps hinzufügen und entfernen. Klicken Sie auf den Link für den benutzerdefinierten Feldnamen, klicken Sie auf das Symbol zum Löschen () mit dem Wert verknüpft und klicken Sie dann auf Feld speichern.

Wenn ein Listenwert entfernt wird, behalten alle Videos, denen dieser Wert zuvor zugewiesen wurde, den Wert. Wenn jedoch ein Video mit dem gelöschten Wert bearbeitet wird, wird der Wert gelöscht und ein neuer Wert muss ausgewählt werden.

Entfernen von benutzerdefinierten Feldern aus einem Konto

Um ein benutzerdefiniertes Feld aus Ihrem Konto zu entfernen, müssen Sie Wenden Sie sich an den Brightcove-Support. Beachten Sie, dass sich das Änderungsdatum für alle Videos ändert, in denen ein Wert vorhanden war, wenn Sie vom Brightcove-Support benutzerdefinierte Felder löschen lassen.

Verwenden von Listenwerten für Brightcove Beacon-Metadaten

In der folgenden Tabelle finden Sie eine Reihe von benutzerdefinierten Feldern, die Sie erstellen müssen. Wie oben gezeigt, wählen Sie beim Erstellen eines benutzerdefinierten Felds den Datentyp entweder Text oder Aufführen. Beachten Sie, dass ALLE Typen sind Text. Es kann den Kuratoren von Videoinhalten entlasten, wenn sie aus einer Liste auswählen können, anstatt Text eingeben zu müssen. Außerdem hilft es unweigerlich, die Dateneingabe konsistenter zu halten, wenn Werte aus einer Liste ausgewählt werden können, anstatt sie einzugeben. Wenn Sie also ein finden Aufführen in einigen Geschäftsfällen funktioniert, kann eine Liste verwendet werden.

Natürlich haben Sie die Wahl, aber Sie können für einige benutzerdefinierte Felder a Aufführen ist eine gute Auswahl für den Datentyp. Beispielsweise können Sie für Listen unter anderem Folgendes berücksichtigen:

  • beacon.genre
  • beacon.ageRating
  • beacon.viewerScore
  • Beacon.rights.<Zähler>.Typ

Benutzerdefinierte Feldnamen

Die nächsten drei Abschnitte dieses Dokuments behandeln benutzerdefinierte Felder, die Sie möglicherweise basierend auf Ihrem speziellen Anwendungsfall für die Implementierung erstellen müssen. Jedes benutzerdefinierte Feld hat beide a Anzeigename und ein Interner Name. Wenn Sie das Gefühl haben, einen aussagekräftigeren Namen für die Anzeigename Option können Sie das verwenden, aber die Interner Name MUST genau übereinstimmen.

Felder für die Rechteverwaltung

Um die Monetarisierungs- und Zugriffsbeschränkungen für Assets zu unterstützen, sollte eine Reihe von benutzerdefinierten Feldern verwendet werden. Da Sie mehrere Instanzen von Rechteverwaltungskonfigurationen haben können, besteht der empfohlene Ansatz darin, mehrere Sätze benutzerdefinierter Felder zu erstellen. Im Namen jeder Instanz wird ein Zähler verwendet, um zwischen ihnen zu unterscheiden. Die erste Instanz sollte den Zähler auf 0 setzen. Hier ist ein Beispiel für diese gruppierten Felder, die tatsächlich einen Zählerwert verwenden:

Beispielsweise benötigen Sie möglicherweise einen Satz, um Ihre in Videos verwendete Werbung zu definieren, und einen anderen, um Sonderangebote für Feiertage wie Silvester zu definieren.

Ein konkretes Beispiel soll die Verwendung dieser Felder verdeutlichen. Der folgende Screenshot zeigt die Einstellung der serverseitigen Anzeigeneinfügung (SSAI), die für ein Video von Video Cloud verwendet wird.

Aus diesem Beispiel können Sie Folgendes lernen:

  • Nicht allen Rechteverwaltungsfeldern einer Gruppe müssen Werte zugewiesen werden. In diesem Fall nur die beacon.rights.<counter>.adConfiguration und Beacon.rights.<Zähler>.Typ werden verwendet.
  • Sie können die in Studio erstellte SSAI-Konfigurations-ID in den benutzerdefinierten Feldern von Brightcove Beacon verwenden.
  • Der Zählerwert für einen Satz von Rechteverwaltungsfeldern kann bei 0 beginnen (gelb hervorgehoben).
  • Obwohl in diesem Screenshot nicht direkt sichtbar, aber wahr ist, dass die Video Cloud-SSAI-Konfiguration in Brightcove Beacon aufgenommen wird.

Beachten Sie, dass keine von Ihnen erstellten Felder die Machen Sie dieses Feld als erforderlich Kontrollkästchen aktiviert.
