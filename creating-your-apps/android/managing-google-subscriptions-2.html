Einführung

Dieses Dokument führt Sie durch die Verwendung der Google Cloud Platform zum Erstellen eines Pub/Sub-Thema und fügen Sie dem Pub/Sub-Thema ein zuvor erstelltes Dienstkonto hinzu.

Pub/Sub-Thema erstellen

Führen Sie die folgenden Schritte aus, um ein Pub/Sub-Thema zu erstellen:

  1. Navigieren Sie zur Indexseite für die Google Cloud-Plattform (https://cloud.google.com).
  2. Drücke den Konsole Link oben rechts auf dem Bildschirm.
  3. Wählen Sie das richtige Projekt aus. Sie haben Ihr Projekt wahrscheinlich nach Ihrer App benannt, z. B. Leuchtfeuer.
  4. In dem Suche , Eintreten Kneipe, und sobald es angezeigt wird, klicken Sie auf Pub/Sub.
  5. Von dem Pub/Sub Seite, klick THEMA ERSTELLEN.
  6. In dem Erstellen Sie ein Thema Dialog, geben Sie die Themen-ID zu sein Untertitel, dann klick THEMA ERSTELLEN.
  7. Klicken Sie auf der angezeigten Seite auf ←, um zum Themen Seite.

Mitglied zum Thema hinzufügen

Führen Sie die folgenden Schritte aus, um ein Dienstkonto als Mitglied des Pub/Sub-Themas hinzuzufügen:

  1. Von dem Themen Seite, für die Untertitel Thema, klick Mehr Aktionen (die drei vertikalen Punkte) dann aus der Dropdown-Liste auswählen Berechtigungen anzeigen.
  2. Klicken Sie im Berechtigungsfeld auf MITGLIED HINZUFÜGEN.
  3. Bestätigen Sie, dass Sie das folgende Formular sehen:
  4. Um die erforderliche E-Mail-Adresse zu erhalten, navigieren Sie zu Google Cloud Console – Dienstkonten Seite. Das von Ihnen erstellte Dienstkonto finden Sie im Verwalten von Google-Abonnements Teil 1 dokumentieren. Kopieren Sie die E-Mail-Adresse für die Android-Abonnements (oder der von Ihnen gewählte Name) Dienstkonto.
  5. Gehe zurück zum Mitglieder hinzufügen Formular in Google Cloud-Plattform.
  6. Fügen Sie die abgerufene E-Mail-Adresse in das Neue Mitglieder Textfeld.
  7. Wählen Sie für die Rolle Pub/Sub → Pub/Sub-Publisher.
  8. Drücke den Benachrichtigungs-E-Mail senden Kontrollkästchen und fügen Sie eine Nachricht hinzu.
  9. Bestätigen Sie, dass Ihr Formular ähnlich wie folgt ausgefüllt ist:
  10. Klicken Speichern.

  11. Als Nächstes fügen Sie eine weitere Mitglieds-E-Mail hinzu. Dadurch wird sichergestellt, dass Google Play-Benachrichtigungen empfangen werden

    Klicken Sie im Berechtigungsfeld auf MITGLIED HINZUFÜGEN.
  12. Bestätigen Sie, dass Sie das folgende Formular sehen:
  13. Fügen Sie die folgende Mitglieds-E-Mail hinzu: 
    google-play-developer-notifications@system.gserviceaccount.com
  14. Stellen Sie die Rolle auf Pub/Sub-Bearbeiter.
  15. Wählen Sie als Nächstes Abonnements.
  16. Auswählen Abonnement erstellen.
  17. Geben Sie ein Abonnement-ID Wert von Untertitel. Von dem Wählen Sie ein Cloud Pub/Sub-Thema aus Wählen Sie im Dropdown-Menü ein Thema aus.
  18. Klicken Schaffen.
  19. Von dem Themen Seite, klicken Sie auf die Untertitel Thema.
  20. Von dem Themendetails , Kopiere das Themenname (Beachten Sie, dass es eine eingebaute Kopiertaste gibt, die rot umrandet ist).
  21. Senden Sie diesen Namen an Ihren Onboarding-Manager.

Überprüfen Sie Ihre Einrichtung

Dieser Abschnitt wird normalerweise vom Brightcove-Onboarding- oder CMS-Team ausgefüllt.

Führen Sie diese Schritte aus, um die Benachrichtigungen für Ihren Publisher-/Abonnenten-Datenaustausch zu überprüfen.

  1. Navigieren Sie im Browser zur Google Play-Konsole (https://play.google.com/console/).
  2. Suchen nach Kneipen.
  3. Auswählen Monetarisierung.
  4. Wählen Sie Ihre App.
  5. Geben Sie das Thema ein.
  6. Auswählen Testbenachrichtigung senden.
  7. Wenn Ihr Setup korrekt ist, sollten Sie Folgendes sehen. Beachten Sie, dass die Weitergabe von Konfigurationen in der Google Cloud einige Zeit in Anspruch nimmt.
